◆ HandleSFMLEvents()

void mbe::StateManager::HandleSFMLEvents ( const sf::Event &  event)

Calls the HandleSFMLEvents() methods of the states currently in the state stack.

The top-most states's method is called first. The HandlerSFMLEvents() methods of the states below are called as long as the HandleSFMLEvents() method of the state above returns true. Therefore, returning false in the state's HandleSFMLEvents() method blocks the events from reaching states below it on the stack.

Parameters
eventThe SFML event containing the event data

◆ Render()

void mbe::StateManager::Render ( )

Calls the Render() methods of the states currently on the state stack.

The top-most states's method is called first. The Render() methods of the states below are called as long as the Render() method of the state above returns true. Therefore, returning false in the state's Render() method blocks the states below from being rendered.
