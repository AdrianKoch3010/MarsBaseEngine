Detailed Description

Acts as a unique identifier for and smart pointer to a const Entity.

Provides a wrapper around an unsigned long long base type. For performance reasons, a pointer to the referenced entity is cached. The entity can be accessed through the interface provided by this class and const correctness will be preserved. If the entity gets deleted is no longer active, the HandleID will no longer be valid.

Note
This is the id type returned by Entity::GetHandleID() when called on a const entity. For a non-const Entity see HandleID<Entity>
See also
HandleBase, Entity, HandleID<Entity>

◆ UnderlyingType

typedef unsigned long long mbe::HandleID< const Entity >::UnderlyingType

Defines the id type used to uniquely identify a handled object.

unsigned long long int (uint64_t) is the largest int type in C++. It allows for 18,446,744,073,709,551,615 unique ids. This is very unlikely to ever be reached.
